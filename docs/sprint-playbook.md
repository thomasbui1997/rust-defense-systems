# Sprint Playbook

## Rules
- 1 ticket per weekday
- weekends = optional deep work
- plan 60–70% capacity

## When stuck
- break task smaller
- check docs
- ask AI one focused question
- switch to fallback ticket

## Ticket types
- learning
- build
- test
- refactor
- fallback
- deep-work

## Definition of done
- compiles
- works
- you understand it
