# AI Usage Guide

Use AI as a learning accelerator, not a substitute for thinking.

## Good uses
- explain a Rust compiler error in plain English
- compare two Rust approaches and tradeoffs
- review a function you already wrote
- suggest smaller subproblems when a task feels too large
- suggest test cases for parser logic

## Bad uses
- generating an entire feature before you attempt it
- copying code you do not understand
- asking for a full project solution up front

## Recommended prompts
- Explain this borrow checker error in plain English and show two fixes.
- Review this Rust function for clarity, ownership, and unnecessary cloning.
- Break this task into smaller steps that each fit in 20–30 minutes.
- Suggest edge cases for this parser.

## Rule of thumb
If you are stuck for more than 10–15 minutes, ask one focused question, then resume coding.
