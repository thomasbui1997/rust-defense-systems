# Rust Defense Systems

A structured, project-driven Rust learning program focused on defense/aerospace-adjacent systems software.

## Purpose
This repository is designed to help me become proficient in Rust through realistic, defense-relevant projects that emphasize:
- systems programming fundamentals
- parsing and validation of device telemetry/logs
- concurrency and networking
- embedded-oriented design constraints
- disciplined engineering workflow using small sprint-sized tickets

## Current focus
### Phase 1 — TACLANE Log Ingestion CLI
A Rust command-line tool for parsing, filtering, validating, and summarizing synthetic TACLANE-style device logs.

## Repository structure
- `docs/roadmap.md` — multi-phase roadmap
- `docs/sprint-playbook.md` — how to work soft sprints and tickets
- `docs/ai-usage-guide.md` — how to use AI as a learning accelerator
- `docs/sprints/sprint-01.md` — first sprint plan and ticket board
- `phase-1-taclane-log-ingestion-cli/` — first Rust project

## Operating model
- 1-week soft sprints
- 5–6 planned tickets per sprint
- weekday tasks sized to about 30–45 minutes
- extra weekend capacity available for deep work
- built-in buffer for real-life interruptions

## Definition of success
Consistency matters more than speed.

The goal is not to rush through Rust, but to steadily build systems-level skill and a portfolio of projects worth discussing in future defense and aerospace interviews.
